## Quantitative Investment Model

**본인의 실전 투자 판단을 돕기 위한 올인원 Python/웹 퀀트 툴**입니다.  
히스토리컬 가격과 팩터 데이터를 자동으로 수집하고, 수익·위험 지표·팩터 노출도를 한 화면에서 확인할 수 있어 **개인이 직접 실전에서 활용**하기 좋게 구성되어 있습니다.

> ⚠️ **면책**  
> 이 도구는 **투자 자문·매매 권유가 아니며**, 출력 결과에 따른 매수·매도 등 모든 의사결정과 손익은 **사용자 본인 책임**입니다. 과거 데이터와 가정에 기반한 분석일 뿐, 미래 수익을 보장하지 않습니다.

---

## 이 도구가 제공하는 가치

- **실전에서 바로 쓰는 지표**: **변동성·샤프 비율·최대 낙폭·VaR·Fama‑French 팩터**를 한 앱에서 계산해, 종목·기간 비교와 리스크 점검에 활용할 수 있습니다.
- **현실 수익률 비교(선택)**: **거래비용·슬리피지·스프레드·세금**을 넣으면 세후·비용 반영 수익률과 샤프 비율로, 실제 수익에 가까운 비교가 가능합니다.
- **다양한 데이터 소스**: yfinance 외 Alpha Vantage, FMP, EODHD, Polygon, Finnhub 등 여러 프로바이더를 골라 쓰며, 티커·기간만 지정하면 자동으로 수집합니다.
- **웹 + CLI**: 브라우저에서 대시보드로 쓰거나, 터미널/스크립트로 반복 분석·자동화할 수 있습니다.
- **개인 맞춤 확장**: 함수 단위 구조로, 본인 전략·팩터·포트폴리오 로직을 붙여 **나만의 실전 툴**로 확장하기 좋습니다.

## 핵심 기능 하이라이트

- **데이터 소스 선택**: 아래 중 하나를 골라 일별 조정 종가(또는 종가)를 수집합니다.
  - **yfinance** — API 키 불필요
  - **Alpha Vantage** — `ALPHAVANTAGE_API_KEY`
  - **Financial Modeling Prep** — `FMP_API_KEY`
  - **EODHD** — `EODHD_API_KEY`
  - **Polygon.io** — `POLYGON_API_KEY`
  - **Finnhub** — `FINNHUB_API_KEY`
  - **Barchart** — `BARCHART_API_KEY`
  - **Quodd** — `QUODD_API_KEY` (REST 스펙은 문서 참고)
  - **Databento** — `DATABENTO_API_KEY`
- API 키는 **환경 변수** 또는 웹/CLI에서 입력해 사용합니다.
- 티커별로 선택한 프로바이더로 요청해 **구조를 맞춰 병합**하고, 결측이 많은 티커는 제거합니다.

### 2. 수익률·리스크 분석

- **일별 수익률 계산**: \( r_t = \frac{P_t}{P_{t-1}} - 1 \)
- **누적 수익률**: \( (1 + r_t) \)를 기간 전체 곱해 성장 곡선 도출
- **연율화 수익률**: 거래일 수(252일)를 기준으로 성장률을 연간 기준으로 변환
- **연율화 변동성**: 일별 수익률 표준편차 × \( \sqrt{252} \)
- **샤프 비율**:  
  \[
  \text{Sharpe} = \frac{\text{연율화 수익률} - \text{무위험 수익률}}{\text{연율화 변동성}}
  \]
- **최대 낙폭(Max Drawdown)**: 피크 대비 최대 골(peak‑to‑trough) 하락률 계산

이 모든 지표를 **티커별 테이블**로 한 눈에 비교할 수 있습니다.

### 2-1. 비용·세금·미시구조 반영 (선택)

- **거래비용(transaction cost)**: 편도 비용(%)을 입력하면 진입·청산 시 각 1회씩 반영합니다.
- **슬리피지(slippage)**: 체결가와 기대가 차이를 편도 %로 반영합니다.
- **스프레드/미시구조(spread)**: bid-ask 스프레드 등 편도 비용을 %로 넣을 수 있습니다.
- **세금(tax)**: 양의 수익에 대해 지정한 세율을 적용한 **세후 누적·연율화 수익률**과 **세후 샤프 비율**을 계산합니다.

비용·세금을 넣으면 테이블에 **누적 수익률(비용·세금 반영)**, **연율화 수익률(비용·세금 반영)**, **샤프 비율(비용·세금 반영)**, **최대 낙폭(비용 반영)** 컬럼이 추가됩니다. 기본값은 모두 0이라, 넣지 않으면 기존과 동일하게 **비반영** 결과만 나옵니다.

### 3. 파라메트릭 VaR (Value‑at‑Risk)

- 사용자가 선택한 신뢰수준(예: 90% / 95% / 99%)에서  
  **정규분포 가정** 하의 일별 VaR(최대 예상 손실)을 계산합니다.
- 포트폴리오/자산의 **하방 리스크를 숫자로** 잡아두고, 실전 리스크 관리에 참고할 수 있습니다.

### 4. Fama‑French 3‑팩터 모델

- Ken French Data Library에서 **일별 3‑팩터(Mkt‑RF, SMB, HML, RF)** 자동 다운로드
- 각 자산에 대해 다음 회귀식을 적합:
  \[
  R_i - R_f = \\alpha + \\beta_{Mkt}(Mkt-RF) + \\beta_{SMB}SMB + \\beta_{HML}HML + \\varepsilon
  \]
- 산출 값:
  - **α (알파)**: 팩터로 설명되지 않는 초과 수익
  - **β 계수들**: 시장, 사이즈, 가치 팩터에 대한 민감도
- `statsmodels` OLS 회귀 결과를 **전문 리포트 형태(summary)** 로 출력
- 팩터별 β를 **막대 그래프**로 시각화하여,  
  “이 종목/포트폴리오가 어떤 팩터에 얼마나 노출되어 있는지” 직관적으로 파악 가능

### 5. 웹 기반 대시보드

- **Streamlit** 기반 모던 웹 UI
  - 사이드바에서 티커, 날짜, 무위험 수익률, **비용·세금·미시구조(선택)**, VaR 옵션, 팩터 모델 옵션을 직관적으로 설정
  - 메인 화면에서:
    - 수익률·리스크 요약 테이블
    - VaR 테이블
    - 가격 시계열(기준일=100), 누적 수익률
    - 팩터 β 막대 그래프, 샤프 비율 차트
    - 티커별 Fama‑French 회귀 요약 (접이식 상세보기)
- 별도의 자바스크립트/프론트엔드 없이 **순수 Python 코드만으로 웹 리포트 구현**

### 6. CLI(커맨드라인) 분석 모드

- 연구·자동화 스크립트 작성에 유용한 **CLI 버전** 제공:
  - `input()` 기반으로 티커/기간/무위험 수익률/옵션 입력
  - 동일한 계산 로직을 터미널 환경에서 그대로 실행

## 예시 워크플로우

1. **티커 입력**: `AAPL, MSFT, SPY`
2. **기간 설정**: `2020-01-01` ~ `2024-12-31`
3. **무위험 수익률**: `0.03` (3%)
4. **옵션 선택**:
   - 비용·세금 반영: 거래비용 0.1%, 슬리피지 0.05%, 세율 25% 등 (선택)
   - VaR 계산: ON, 신뢰수준 95%
   - Fama‑French 3‑팩터 분석: ON
5. **결과 해석**:
   - 어떤 자산이 높은 **위험 대비 보상(샤프 비율)** 을 갖는지 비교
   - **최대 낙폭**을 통해 하락장 민감도 파악
   - **팩터 β**를 통해 “이 포트폴리오는 성장/가치/소형주 어느 쪽에 더 베팅되어 있는가?” 분석

## 활용 아이디어

- **실전 종목·기간 비교**: 관심 종목/ETF의 수익률·변동성·샤프·최대 낙폭을 기간별로 비교해 포지셔닝 참고
- **비용·세금 반영 수익률**: 거래비용·세금을 넣어 실제 체감 수익에 가까운 숫자로 의사결정 보조
- **팩터 노출 점검**: Fama‑French β로 포트폴리오가 시장·가치·사이즈에 얼마나 노출돼 있는지 확인
- **리스크 한도 설정**: VaR·최대 낙폭으로 손실 구간을 가늠하고, 본인 리스크 한도에 맞춰 조정

## 한계와 주의사항

- **비용·세금·미시구조**: 거래비용·슬리피지·스프레드·세금은 옵션 입력 시 반영되며, 편도 비용 합산·진입/청산 각 1회, 세금은 양의 수익에 대한 단일 세율의 간이 모형입니다. 실시간 유동성·시장충격 등 정교한 미시구조는 반영되지 않습니다.
- VaR·팩터 모델은 **역사적 데이터와 단순 가정(정규분포, 선형 관계 등)** 에 기반합니다.
- **과거 성과는 미래 수익을 보장하지 않으며**, 이 도구의 결과는 투자 손실에 대한 보증이 아닙니다. 실전 매매·포지션 결정은 전적으로 **사용자 본인 판단과 책임**입니다.

이 프로젝트는 **개인이 실전에서 직접 쓰는 정량 분석 툴**로 쓸 수 있도록 구성되어 있습니다.  
비용·세금을 반영해 현실에 가까운 비교를 하고, 필요하면 팩터·포트폴리오·백테스트 로직을 붙여 **나만의 실전 퀀트 툴**로 확장해 사용할 수 있습니다.
